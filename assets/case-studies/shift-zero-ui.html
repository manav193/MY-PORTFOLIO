SHIFT-ZERO UI

A dedicated presentation-layer and UI/UX exploration for SHIFT-ZERO, separated from the gameplay repository so visual systems can evolve without conflating them with game logic.

Challenge

Design a high-contrast interface language that stays readable during fast gameplay while scaling across menus, HUD, progression, shop, missions, collections, statistics, rewards, and settings.

System

The work establishes reusable visual patterns and screen hierarchy for main menu, HUD, shop, daily missions, achievements, collection book, statistics, daily login, lucky spin, chests, and settings.

Implementation principle

The UI direction was applied to the Godot presentation layer while protecting gameplay architecture. That separation reduces regression risk and keeps design iteration independent from core rules and services.
